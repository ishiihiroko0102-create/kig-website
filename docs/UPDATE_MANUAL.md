## 1. 基本概念 - 5分で理解

### ホームページの構成

```
ホームページ
 ├── トップページ（/）
 ├── 企業情報ページ（/about）
 ├── 採用情報ページ（/recruit）
 ├── ニュース・ブログ（/news）
 ├── お問い合わせ（/contact）
 └── その他ページ
```

**各ページの中身は、以下で構成されます：**

```
ページ
 ├── ヘッダー（最上部：ロゴ + メニュー）
 ├── メインコンテンツ（ページ固有の内容）
 │   ├── セクション 1（テーマ別の区切り）
 │   ├── セクション 2
 │   └── セクション 3
 └── フッター（最下部：会社情報 + リンク）
```

### 修正の基本ステップ

```
1️⃣ 修正したいページ・セクションを特定する
   例）「採用ページの給与情報を更新したい」

2️⃣ 修正対象ファイルを開く
   例）src/pages/recruit.astro

3️⃣ 該当テキストを探して修正
   例）「月給 250,000円」→「月給 280,000円」

4️⃣ 変更を保存
   Git にコミット（変更を記録）

5️⃣ デプロイ
   自動的にホームページに反映
```

## 2. よくある更新作業 - 選んで実行

### 2.1 テキスト内容を修正・更新する

**例）採用ページの職種説明を更新したい**

#### ステップ1：修正対象を特定

```
修正箇所：採用ページ
ファイル：src/pages/recruit.astro
セクション：職種説明
```

#### ステップ2：ファイルを開く

```bash
$ 用意されたエディタで以下を開く
  src/pages/recruit.astro
```

#### ステップ3：該当テキストを探す

**修正前の例：**
```html
<!-- 営業スタッフ の説明 -->
<div class="job-description">
  <h3>営業スタッフ</h3>
  <p>中古自動車パーツの買取・販売営業</p>
  <p>給与：月給 250,000〜600,000円</p>
</div>
```

#### ステップ4：テキストを修正

**修正後の例：**
```html
<!-- 営業スタッフ の説明 -->
<div class="job-description">
  <h3>営業スタッフ</h3>
  <p>中古自動車パーツの買取・販売営業</p>
  <p>給与：月給 280,000〜650,000円</p>  ← ここを修正
</div>
```

#### ステップ5：変更を保存・コミット

```bash
$ git add src/pages/recruit.astro
$ git commit -m "fix: 営業スタッフ給与を更新（250k→280k）"
$ git push
```

**⏱️ 所要時間：5分**

### 2.2 ニュース・ブログ記事を追加する

**例）「2月の採用実績」を公開したい**

#### ステップ1：テンプレートファイルを複製

```bash
# テンプレートコピー
$ cp src/pages/news/template.md src/pages/news/2026-02-21-hiring-results.md
```

#### ステップ2：記事の情報を入力

```markdown
---
title: 2月度 採用実績レポート
date: 2026-02-21
category: 採用
author: 石井 浩子
image: /images/news/feb-hiring.jpg
---

## 2月度採用実績

先月、以下の職種で計5名の採用が決定しました。

### 採用内容
- 営業スタッフ：2名
- 倉庫内物流スタッフ：2名
- データ入力スタッフ：1名

### スタッフの声
> 新しいチームメンバーの加入により、さらに体制が強化されました。

---
```

#### ステップ3：画像を準備（必要に応じて）

```
/public/images/news/
  └── feb-hiring.jpg （1200x600px推奨）
```

#### ステップ4：変更を保存・コミット

```bash
$ git add src/pages/news/2026-02-21-hiring-results.md
$ git add public/images/news/feb-hiring.jpg
$ git commit -m "feat: 2月採用実績ニュース記事追加"
$ git push
```

**⏱️ 所要時間：15-20分**

### 2.3 SNS リンク・埋め込みを更新する

**例）「Instagram フォロー数が増えたので、画面に表示する」**

#### ステップ1：修正対象を特定

```

```

#### ステップ2：既存の Instagram リンクを確認・更新

**修正前：**
```html

  Instagram を見る
</a>
```

**修正後（例）：フォロワー数を表示追加**
```html

  Instagram を見る
  <span class="follower-count">（フォロワー数：5.2k）</span>
</a>
```

#### ステップ3：変更を保存・コミット

```bash
$ git add src/pages/index.astro
$ git commit -m "feat: InstagramフォロワーカウントをUI に追加"
$ git push
```

**⏱️ 所要時間：10分**

### 2.4 イベント写真アルバムを追加する

**例）「3月のイベント出展写真を公開したい」**

#### ステップ1：アルバムテンプレートを作成

```
/src/pages/events/2026-03-spring-event.astro
```

#### ステップ2：基本構造を記入

```astro
---
// ページの設定
const title = "春の自動車フェスタ 2026 出展レポート";
const date = "2026-03-15";
const location = "大阪メッセ";
const images = [
  { src: '/images/events/event01.jpg', alt: 'スタッフと来場者' },
  { src: '/images/events/event02.jpg', alt: '商品展示' },
  { src: '/images/events/event03.jpg', alt: 'スタッフ集合写真' },
];
---

<div class="event-album">
  <h1>{title}</h1>
  <p>開催日：{date} | 場所：{location}</p>

  <div class="gallery">
    {images.map(img => (
      <img src={img.src} alt={img.alt} />
    ))}
  </div>
</div>
```

#### ステップ3：画像ファイルを準備

```
/public/images/events/
  ├── event01.jpg
  ├── event02.jpg
  └── event03.jpg
```

#### ステップ4：変更を保存・コミット

```bash
$ git add src/pages/events/2026-03-spring-event.astro
$ git add public/images/events/
$ git commit -m "feat: 3月イベント出展写真アルバム追加"
$ git push
```

**⏱️ 所要時間：20-30分**

## 3. コンテンツ作成ガイドライン

### 3.1 文体・表現

#### ✅ 推奨する表現

```
「当社は、中古自動車パーツの買取・販売を専門とする企業です。」
→ ビジネス的で信頼感がある

「お客様とのパートナーシップを大切にします。」
→ 企業理念に沿っている
```

#### ❌ 避けるべき表現

```
「パーツを安く買ったり売ったりしてます」
→ 粗雑で企業としての信頼感が低い

「こんなパーツも扱ってます！」
→ カジュアルすぎて、信用調査対応に不適切
```

---

### 3.2 数字・価格の表記

#### ✅ 推奨

```
給与：月給 250,000～600,000円（試用期間中：230,000円）
設立：2010年9月29日
資本金：8,000万円
```

#### ❌ 避けるべき

```
給与：月給 25万～60万くらい（信用調査で信頼性低下）
設立：2010年（正確な年月を明記すべき）
売上：たくさん（具体性がなく、信用調査に対応できない）
```

---

### 3.3 スタッフ紹介時のプライバシー

#### ✅ 掲載してよい

```
名前：石井 浩子
役職：代表取締役
プロフィール：「チーム全体を支える」
顔写真：会社公式撮影
```

#### ❌ 掲載しない

```
住所（東京都渋谷区△△）
プライベート SNS アカウント
自宅電話番号
個人的な趣味・特技（例：猫好き、バイク好き等）
```

---

### 3.4 画像のサイズ・形式

| 用途 | サイズ | 形式 | 圧縮 |
|------|--------|------|------|
| アイキャッチ画像 | 1200 × 600 px | JPG / WebP | 圧縮推奨 |
| スタッフプロフィール | 500 × 500 px | JPG / PNG | 軽め |
| イベント写真 | 1920 × 1280 px | JPG | 圧縮推奨 |
| SNS用 | 1080 × 1080 px（Instagram） | JPG / PNG | 軽め |

**After（AI が生成）：**
```
「当社は、自動車パーツの買取・販売を専門とし、
市場最適価格での取引を実現します。
顧客・パートナーと長期的な信頼関係を構築することで、
```

## 5. よくある質問（FAQ）

### Q1: 「修正を失敗したらどうしよう？」

**A:** 心配不要です。Git で全ての変更履歴が記録されるため、いつでも前の版に戻せます。

```bash
# 失敗時は前の版に戻す
$ git revert <commit番号>
```

---

### Q2: 「HTML タグってどこまで修正していいの？」

**A:** 基本的には **テキストの中身だけ** を修正してください。

✅ OK：テキスト内容の修正
```html
<p>給与：月給 250,000円</p>  →  月給 280,000円 に修正
```

❌ NG：タグ構造の削除・追加
```html
<p>を削除したり、<div>を追加したりしない
```

---

### Q3: 「新しいセクション追加したい場合は？」

**A:** その場合は、AI または開発者に相談してください。

テンプレート化して「初心者向け」に用意することで、次回から簡単に追加できるようにします。

---

### Q4: 「画像のサイズ・形式がよく分からない」

**A:** 以下のツールで簡単に調整できます：

- **画像圧縮**: TinyPNG（https://tinypng.com/）
- **サイズ変更**: Canva（https://canva.com/）
- **形式変換**: CloudConvert（https://cloudconvert.com/）

## 6. トラブルシューティング

### 「デプロイしたのに反映されない」

**原因：** キャッシュ問題

**解決方法：**
1. ブラウザを再読み込み（Ctrl+F5 または Cmd+Shift+R）
2. 5～10分待つ（Vercel が反映に時間かかる場合）
3. それでもダメなら、開発者に連絡

---

### 「修正したテキストが文字化けしてる」

**原因：** ファイルの文字コード設定

**解決方法：**
1. ファイルを開く際に「UTF-8」を選択
2. 再度修正して保存

---

### 「画像がアップロードできない」

**原因：** ファイルサイズが大きい

**解決方法：**
1. TinyPNG で圧縮
2. ファイルサイズを 2MB 以下に
3. 再度アップロード
