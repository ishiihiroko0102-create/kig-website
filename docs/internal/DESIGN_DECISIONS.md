## 1. コンポーネント設計の判断

### 1.1 採用ページの職種カード設計

**決定日**: 2026年2月20日
**決定者**: 石井 浩子 + Claude Code

#### 背景・課題
- 3職種（営業、物流、データ入力）を分かりやすく表示したい
- BtoB営業先にもアピールしたい
- モバイルでも見やすくしたい

#### 検討案
1. **案A**: 職種ごとに詳細ページを作成
   - メリット：詳細情報を充実できる
   - デメリット：ページが増えて管理が大変

2. **案B**: カード形式でまとめて表示（採用）
   - メリット：一覧性が高い、モバイル対応が簡単
   - デメリット：詳細情報は限定的

#### 採用理由
**案B を採用** - 初期段階は「採用職種をシンプルに見せる」ことを優先

## 2. コンテンツ開示基準の判断

### 2.1 給与情報の詳細度

**決定日**: 2026年2月21日
**決定者**: 石井 浩子

#### 記録内容

| 情報 | 掲載判断 | 理由 |
|------|--------|------|
| 基本給（月給範囲） | ✅ 掲載 | 採用競争力のため、必須 |
| 試用期間中の給与 | ✅ 掲載 | 透明性が信用につながる |
| ボーナス | 🔄 検討中 | 実績によって変動するため要確認 |
| 手当詳細（交通費等） | ❌ 掲載しない | 税務署から質問される可能性 |
| 過去の給与改定例 | ❌ 掲載しない | 経営情報として秘密保持 |

#### AI 活用記録

**使用プロンプト:**
```
給与情報のホームページ掲載について、以下の観点から確認してください：

1. 採用競争力（求職者にアピール）
2. 信用調査対応（営業先の信頼）
3. 税務署対応（脱税疑い回避）
4. 経営秘密保護（競合他社への情報漏洩防止）

給与情報の掲載基準を提案してください。
```

**AI の回答**: （記録例）
```
推奨開示内容：
- 月給の範囲（額面表記）
- 試用期間中の給与（差額明記で透明性確保）

非推奨開示内容：
- 総支給額の詳細（ボーナス込み等）
- 手当の種類・額（内訳から脱税疑いを招く可能性）
- 給与改定の実績（経営情報として秘密保持が無難）
```

## 3. SNS・コンテンツ戦略の判断

### 3.1 Instagram フィード埋め込みの判断

**決定日**: 2026年2月21日
**決定者**: 石井 浩子

#### 背景
- Instagram で週1-2回投稿中
- ホームページに Instagram フィードを表示したい
- リアルタイムでコンテンツが更新される利点

#### 検討結果

✅ **埋め込みを実装する**

**理由:**
1. ホームページとSNSの連携で「活発な企業」感を演出
2. 手動更新の手間が削減される
3. Instagram フォロワー増加につながる
4. モバイルユーザーに届きやすい

**実装方法:**
- Instagram の公式 API を使用
- トップページの「最新情報」セクション
- 最新3～5投稿を表示

## 4. セキュリティ・プライバシーの判断

### 4.1 メンバーページの認証方式

**決定日**: 2026年2月21日
**決定者**: 石井 浩子 + セキュリティ要件

#### 検討内容

| 方式 | メリット | デメリット | 採用 |
|------|--------|----------|------|
| **メールアドレス + パスワード** | 一般的で分かりやすい | パスワード管理が課題 | ✅ 検討中 |
| **Google/Microsoft アカウント連携** | 管理が簡単 | 外部依存 | ⏳ 将来対応 |
| **社内 LDAP/Active Directory** | セキュアで一元管理 | 導入コスト高 | ❌ 現段階では不要 |

#### 採用予定

**メール + パスワード方式**を初期段階で実装

**セキュリティ対策:**
- パスワードは 8 文字以上、大文字・数字・記号を含む
- HTTPS 通信
- セッションタイムアウト（30分）
- ログイン試行回数制限

## 5. パフォーマンス最適化の判断

### 5.1 ヒーロー画像の背景動画 vs 画像

**決定日**: 2026年2月20日
**決定者**: Claude Code デザイン決定

#### 検討内容

| 方式 | メリット | デメリット | 採用 |
|------|--------|----------|------|
| **背景動画** | インパクト大、Apple 風 | ファイルサイズ大、再生エラー可能性 | ✅ 採用（最適化必須） |
| **高品質画像** | 軽量、高速読み込み | インパクト劣る | ⏳ フォールバック用 |

#### 採用理由

**背景動画を採用**（ただし最適化が必須）

**最適化方法:**
- MP4 形式（h.264 コーデック）
- 解像度：1920×1080px
- ファイルサイズ：最大 5MB
- 長さ：最大 10 秒
- フォールバック画像：同時掲載

## 6. コンテンツカレンダー・更新頻度の判断

### 6.1 ニュース更新頻度の決定

**決定日**: 2026年2月21日
**決定者**: 石井 浩子

#### 検討プロセス

**現状の投稿実績:**
- Instagram：週 1～2 回
- YouTube：月 1 回

**ホームページへの適用:**
- ニュース記事：月 1～2 回（無理なく継続可能な頻度）
- イベント写真：イベント実施時（不定期）
- スタッフ紹介：新入社員時期のみ

**判断理由:**
- 無理なく継続できる頻度を優先
- 「更新が止まる」より「定期的に少量」が信用につながる
