## 1. ホームページの使命・ビジョン

### 1.1 ミッションステートメント

> 株式会社KIGのホームページは、「多角経営企業としての信頼と存在感」を伝え、
> 全てのステークホルダーに対して最適な情報を届けるデジタル拠点である。

### 1.2 ターゲット別の役割定義

| ターゲット | ホームページの役割 | 到達ページ | 期待するアクション |
|-----------|----------------|----------|----------------|
| **BtoB営業先** | 信用調査に耐える企業情報の提供 | About, Business | 取引開始の検討 |
| **BtoC顧客** | 買取・購入サービスの理解促進 | Top, Business | ECサイトへの遷移・問合せ |
| **求職者** | 魅力的な職場環境の提示 | Recruit, Staff | 応募・問合せ |
| **信用調査機関** | 事業の健全性・規模の証明 | About | 企業評価の向上 |
| **社内スタッフ** | 業務マニュアルへのアクセス | Members | 業務効率化 |

### 1.3 Apple風デザイン実現の方針

SPEC.md に記載の通り、デザインは「白ベース + ネイビーアクセント（MDSファンド風）」で確定済み。
ここでいう「Apple風」とは、以下の要素の適用を指す。

| Apple風要素 | KIGサイトでの実現方法 |
|-----------|-------------------|
| **余白の贅沢な使い方** | セクション間 padding: 120px以上、テキスト周囲に十分な空間 |
| **大胆なタイポグラフィ** | 見出しは48px-72px、ウェイトの対比で視覚的ヒエラルキー |
| **1画面1メッセージ** | 各セクションは1つのメッセージに集中、スクロールで次の話題 |
| **高品質な画像/グラフィック** | 当面はアイコン+タイポグラフィで代替、写真は段階的に追加 |
| **微細なアニメーション** | スクロールフェードイン（確定済み: 0.7s ease, translateY 32px） |
| **フラットで統一的な色使い** | ネイビー(#0d1f6e) + 白(#ffffff) + グレー階調の限定パレット |

### 3.3 認証方式（共通パスワード）

**確定方式:** クライアントサイド共通パスワード認証

**実装方法:**

```
アクセスフロー:
1. /members にアクセス
2. パスワード入力画面を表示
3. 正しいパスワード入力 → Cookie にトークンを保存
4. 以降、Cookie が有効な間はアクセス可能
5. Cookie 有効期限: 30日間

技術実装:
- パスワードは環境変数に格納（MEMBERS_PASSWORD）
- bcrypt でハッシュ化して比較
- Cookie: members_auth=<hashed_token>; max-age=2592000; path=/members
- 全 /members/* ルートで Cookie を検証
```

### 4.3 SNS連携方針

| SNS | 連携方法 | ホームページでの表示 |
|-----|---------|------------------|
| **Instagram** | API埋め込み or 手動リンク | Topページにフィード表示（最新6投稿） |

### 4.4 コンテンツカレンダー案（月4～8件の短編ニュース形式）

**月間テンプレート（4～8記事/月、柔軟運用）:**

| 週 | コンテンツ種別 | 具体例 | 文字数 | 制作時間目安 |
|----|-------------|-------|-------|-----------|
| 第1週 | **新入荷情報** | 注目入荷パーツ紹介 | 50-100語 | 15分 |
| 第2週 | **お知らせ/キャンペーン** | 営業情報、キャンペーン告知 | 80-150語 | 20分 |
| 第3週 | **YouTube or Tips** | 動画紹介 or 豆知識 | 50-150語 | 15分 |
| 第4週 | **スタッフ紹介 or メディア** | YouTube動画紹介 or スタッフ紹介 | 100-130語 | 20分 |
| 臨時 | **速報** | 台風休業、緊急告知 | 30-50語 | 10分 |

**月4件の場合:** 各週1件 = 30分程度（実現可能性 ⭐⭐⭐⭐⭐）
**月8件の場合:** 週2件 = 柔軟に対応（実現可能性 ⭐⭐⭐⭐）

**持続可能性のポイント:**
- 1記事あたり15～20分で作成可能（短編ニュース形式）
- AIプロンプトテンプレートにより、入力情報を渡すだけで記事が生成可能
- 写真は Instagram と共用し、追加撮影不要
- 短編フォーマットで「毎回ゼロから考える」負荷を排除
- 無理のない「月4～8件」ペースを推奨

**詳細テンプレートは CONTENT_DETAILED_STRATEGY.md セクション 5 を参照**

### 9.1 週1回以上のニュース更新方法

**手順:**

```
1. ニュースの内容を決める

   - コンテンツカレンダー（4.4参照）を確認

2. Markdown ファイルを作成
   - ファイル名: YYYY-MM-DD-title.md
   - 保存先: src/content/news/
   - テンプレートをコピーして記入（4.2参照）

3. AIに依頼する場合のプロンプト
   「以下の情報でニュース記事を作成して。
    種別: [商品紹介]
    内容: [新入荷のBMW E90用ヘッドライト]
    ポイント: [状態良好、即日発送可能]
    形式: src/content/news/ のMarkdownテンプレートで」

4. Git で反映
   git add .
   git commit -m "feat: ニュース記事追加 YYYY-MM-DD"
   git push origin main
   → Vercel が自動デプロイ（約1分で反映）
```

**所要時間目安:** 15-30分/記事

### 9.2 メンバーページマニュアル掲載方法

**手順:**

```
1. マニュアルの内容を準備
   - 既存の Word/PDF マニュアルがあればテキスト抽出
   - AIに「このPDFの内容をMarkdown形式に変換して」と依頼

2. Markdown ファイルを作成
   - 保存先: src/content/manuals/[カテゴリ]/
   - フロントマター（3.4参照）を記入
   - 本文を Markdown で記載

3. カテゴリの追加（新カテゴリの場合）
   - src/content/manuals/ にフォルダを追加
   - config.ts を更新（必要な場合）

4. Git で反映（ニュースと同じ手順）
```
